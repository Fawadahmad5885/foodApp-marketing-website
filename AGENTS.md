# AGENTS.md

## Project Overview

This project is a modern marketing website for a food delivery platform inspired by Blink.

The purpose of this website is to help restaurants grow direct online ordering revenue through:

* Branded websites
* Mobile apps
* Delivery management
* Loyalty programs
* Marketing automation
* Customer retention tools

This is NOT the customer ordering application.
This is NOT the restaurant dashboard.

This repository only contains the marketing website.

---

## Technology Stack

* Next.js
* React
* TypeScript
* Tailwind CSS

## Development Principles

Before making any change:

1. Read the relevant files.
2. Understand existing patterns.
3. Follow the current architecture.
4. Reuse existing components whenever possible.

---

## Coding Rules

* Prefer functional components.
* Keep components small and reusable.
* Avoid unnecessary abstractions.
* Do not create duplicate components.
* Use TypeScript types whenever possible.
* Follow existing folder structure.

---

## UI Rules

* Maintain consistent spacing.
* Maintain consistent typography.
* Use responsive layouts.
* Avoid hardcoded widths when possible.
* Keep sections visually balanced.

---

## Content Rules

Marketing copy should:

* Focus on benefits
* Be concise
* Be conversion oriented
* Use simple language
* Avoid buzzwords

Good:

"Increase direct orders and keep more revenue."

Bad:

"Leverage synergistic omnichannel digital transformation."

---

## SEO Rules

When creating pages:

* Use meaningful headings.
* Use semantic HTML.
* Add metadata when appropriate.
* Prioritize Core Web Vitals.

---

## Safety Rules

Never:

* Delete unrelated files.
* Refactor the entire project unless requested.
* Introduce large dependencies without approval.
* Change project architecture without approval.

---

## Command Execution Policy

IMPORTANT:

Do NOT automatically run:

npm run dev

npm run build

npm start

pnpm dev

pnpm build

yarn dev

yarn build

next build

next start

Do not launch localhost.

Do not start servers.

Do not execute terminal commands unless explicitly requested.

If verification is needed:

Explain which command would be useful and ask first.

---

## Required Workflow

For every task:

1. Explain the plan.
2. Identify files to edit.
3. Make the smallest possible change.
4. Summarize completed work.
5. Mention files modified.

Never skip this workflow.
