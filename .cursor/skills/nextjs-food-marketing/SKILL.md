# Purpose

This skill helps build and maintain a high-converting food delivery platform marketing website.

The website promotes:

* Branded websites
* Mobile apps
* Loyalty systems
* Delivery management
* Restaurant growth tools

---

# Expected Workflow

1. Read relevant files.
2. Understand existing patterns.
3. Determine smallest change needed.
4. Implement change.
5. Summarize modifications.

---

# Design Expectations

UI should feel similar to:

* Blink
* Owner.com
* Flipdish
* Toast

Characteristics:

* Clean
* Premium
* Fast
* Conversion focused

---

# Component Guidelines

Prefer:

* Reusable components
* Clear prop names
* Responsive layouts

Avoid:

* Duplicate components
* Large monolithic files
* Unnecessary abstractions

---

# Content Guidelines

Headlines should:

* Explain value quickly
* Focus on benefits
* Be easy to scan

Examples:

Good:
"Launch your branded ordering app."

Good:
"Turn customers into loyal regulars."

Bad:
"Revolutionize your omnichannel ecosystem."

---

# Command Restrictions

Do not run:

npm run dev

npm run build

pnpm dev

pnpm build

yarn dev

yarn build

localhost

development servers

Unless the user explicitly asks.

---

# Success Criteria

Every change should improve at least one:

* User experience
* Conversion rate
* Maintainability
* SEO
* Accessibility
